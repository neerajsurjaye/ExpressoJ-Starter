SHOW ME THE CODE

Here's a sample application that uses the .use() method to intercept requests and handle GET requrest on /hello endpoint. 

public static void main(String[] args) {
    /* Initialize expresso */
    Expresso expresso = Expresso.init();
    /*
    * .use() executes on all requests, regardless of the HTTP method.
    * A path can also be specified.
    */
    expresso.use((req, res, next) -> {
        res.writeResponse("Request intercepted");
        next.next();
    });
    /* Supports [get, post, put, delete] http methods */
    expresso.get("/hello", (req, res, next) -> {
        res.writeResponse("Hello world");
        next.next();
    });
   
    /* Set a listening address */
    expresso.listen(5757);
    /* Start the app */
    expresso.start();
}

USE EXPRESSOJ

Follow the below steps to setup ExpressoJ on your local System.

1. Clone ExpressoJ github repo to your local system.

Github: ExpressoJ repo 

$ git clone https://github.com/neerajsurjaye/ExpressoJ.git

2. Cd into the project and change repository to main

$ cd ~/ExpressoJ
$ git checkout main

3. Build the project

$ mvn clean install

4. Now add the ExpressoJ as dependency in your own project 

<dependencies>
    ...
    <dependency>
        <groupId>com.spec.web</groupId>
        <artifactId>expressoj</artifactId>
        <version>1.0.0-SNAPSHOT</version>
    </dependency>
    ...
</dependencies>

5. In your project update the following properties in pom.xml

  1. Change build type to jar.
  2. Use maven-shade-plugin to make it a uber jar.
  3. (optional) Use maven-jar-plugin to add a manifest to main class. 
<!-- Jar packaging required -->
<packaging>jar</packaging>

<build>
    ...
    <plugins>
        <!-- Maven shade plugin: Required to create uber(fat) 
        JAR that includes all the dependencies -->
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-shade-plugin</artifactId>
            <version>3.2.1</version>
            <executions>
                <execution>
                    <phase>package</phase>
                    <goals>
                        <goal>shade</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>

        <!-- Maven jar plugin: Adds a manifest file specifying 
        the main class. -->
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-jar-plugin</artifactId>
            <version>3.2.0</version>
            <configuration>
                <archive>
                    <manifest>
                        <!-- Change this to fully qualified name of 
                        your main class -->
                        <mainClass>com.spec.Main</mainClass>
                    </manifest>
                </archive>
            </configuration>
        </plugin>
    </plugins>
    ...
</build>
